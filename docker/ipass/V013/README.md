3. 服务器重启、Docker重启，服务自动重启

   某个服务出现问题，紧急更新(版本不升级)  
   某个服务迭代升级，发布新版本(版本升级)  
   部署到一台、两台、三台等不同部署方式  
   客户提供mysql、redis可以直接使用(符合版本要求)

6. 启动端点
